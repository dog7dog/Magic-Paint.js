# 2D Animation JS — 物理UI削除・回転中心・定規ずれ修正版

## 修正内容

1. 物理演算のUIを削除
   - 右パネルの物理ボタン
   - 左プロパティのRigidbody、床、材質、弾性、摩擦、質量、重力
   - 右クリックメニューのRigidbody/床項目

2. 自動回転を図形の中心基準に修正
   - 四角、円、三角、多角形、線、ペン、ブラシの中心を取得
   - Canvas再生とGSAPプレビュー/JS生成の両方で中心回転

3. 定規表示中の二重表示/ずれを修正
   - 古いabsolute配置CSSを削除
   - CanvasとSVGオーバーレイを同じ `cv-wrap` に重ねるよう修正
   - ガイドラインとブラシカーソルもキャンバス領域基準に修正

## 起動

```bash
pip install flask
python app.py
```

ブラウザ:

```text
http://127.0.0.1:5000
```


## 追加修正

- 下の再生/タイムラインUIをドラッグで高さ調整可能にしました。
- タイムラインに「小・中・大・折畳」ボタンを追加しました。
- タイムラインの高さはブラウザに保存されます。
- JSエディタを折りたたむとキャンバス領域が広がります。


## アニメパス安定化

前回の座標変換修正は戻しました。  
今回は座標系を変えず、以下だけを修正しています。

- パス補間を距離ベースに変更
- パス点が細かすぎて暴れないよう間引き
- GSAPプレビューのパスサンプリングを増やして線に近づける


## ボタン復旧版

- 動いていた `animation_js_path_stable` をベースに作り直しました。
- 既存のボタンIDとイベント登録は変更せず、追加機能は後付け初期化にしました。
- グループ化: Shift+クリックで複数選択、Ctrl+G、Ctrl+U。
- FPS変更: 12/24/30/60。
- プレビュー: `openPreview()` のみ安全に差し替え。


## Canvasプレビュー修正

SVG/GSAP MotionPath変換をやめ、プレビューをCanvas再生方式へ変更しました。  
編集画面と同じ座標系で `pos - animPath[0]` を使うため、プレビューで図形が大きく飛ぶ問題を避けます。

注意:
- ブラシのぼかしsnap画像はプレビューでは線として簡易描画します。
- 図形、線、ペン、基本パス、自動回転はCanvas座標で再生されます。


## グループ化

- Shift+クリックで複数選択
- Ctrl+G または上バー「グループ」でグループ化
- Ctrl+U または上バー「解除」でグループ解除
- グループ内の1つを動かすと、同じグループの図形も一緒に移動
- グループ枠を表示
